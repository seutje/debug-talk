Console

  • Standardized by FireBug (Chrome's docs)
  • Command line API
  • jQuery($0);, $_ & inspect() FTW!
  • Per-frame execution and cd() frame traversal
  • Can persist over page-calls

Elements

  • Customizable for great good! (it's just CSS & JS, and that's what we do, right?)
  • (rendered) DOM tree: shows location and dimensions on hover
  • (Computed) Style: new rule, emulate mouse actions, highlights triggering selector
  • Metrics: visual dimensions
  • Properties: prototypal chain
  • DOM breakpoints: break on mutation events
  • Event Listenters: lists handlers on element or entire tree
  • elements are listed

Resources

  • Per-frame and per-type list of assets
  • Cookies
  • Various JavaScript storage APIs

Network

  • Blue: html
  • Green: css
  • Orange: js
  • Purple: images
  • Yellow: json
  • Red: fonts
  • ...
  • Click to inspect headers
  • See live websockets connections

Sources

  • Main JavaScript debugging tool
  • Set breakpoints
  • Step through code bit by bit
  • Monitor variables throughout the flow
  • Pretty print!
  • (Live editing)

Timeline

  • Events: Analyze events triggering costly repaints and blocking operations
  • Frames: Analyze which frames are taking too long to draw and why
  • Memory: Analyze memory usage and garbage collection
  • Timeline viewer

Profiles

  • CPU profile: which functions are costly to run and where exactly are my GHz going
  • CSS selector profile: which selectors aren't used and which are costly
  • Heap snapshot: where is my memory going, track reachable objects

Audits

  • Easy list with possible improvements like unused CSS rules, compression optimization, ...

  • It pays to know what you're debugging, use Wappalyzer if you must
